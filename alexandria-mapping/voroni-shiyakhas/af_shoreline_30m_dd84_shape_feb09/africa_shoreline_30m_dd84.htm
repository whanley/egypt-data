Process_Step:
Process_Description:
The 30-meter Landsat imagery for the shoreline dataset was downloaded from the Global Land Cover Facility's Earth Science Data Interface at <http://glcfapp.umiacs.umd.edu>. A Path/Row search to acquire ETM+ sensor data was completed for each Landsat7 scene covering the islands and coastline of Africa. Bands 4, 3, and 2 were layerstacked and output to an Imagine file using Erdas Imagine software. Water thresholds were determined and recorded for each image using the attribute editor in a display window. When the ETM+ data was partially or completely covered with clouds, additional imagery, including SRTM data, was used to complete the water delineation for each scene. A series of models were compiled to input the respective water thresholds, eliminate extraneous pixels, and produce a two-bit image of a water and land image. Images from the same UTM zones were combined in Imagine. Each combined output was run through an initial QA/QC process to clean up any problem areas primarily related to cloud cover or confusion with shadows or inland water pixels. The raster/vector tools were used to convert the raster image to an Arc vector coverage. Arcs were smoothed, joined, and projected to a geographic decimal degrees projection, WGS84 datum, using ArcMap software. All files were joined into a single vector layer using tools in ArcMap.
Clean up of the extracted Africa coastline vectors was done using Arctools Edit where the goal was to create polygon topology for the continent and outer islands in a single file. Various line work issues were cleaned up including, overshoots, dangles, double lines, edge matching and eliminating extra nodes in the coverage. Some line work was redone at this step to correct for clouds, which obscured the coast, or was mapped as unwanted islands. Removal of any inland lakes was completed as well, which was achieved by a recode process during the next step, converting to a raster file. Once the vector file was cleaned, it was split into 2 halves and an 8-bit raster file was generated using the PolyGRID command. This creates a rather large file, thus splitting up both sides was necessary for a successful output. The next step was to convert the 2 halves to a single 1-bit file, reducing the file size so that the two could be merged together. The resulting raster file is a 1-bit image, with values 1 (land) and 0 (water) for the continent of Africa, including Madagascar and other various islands. Additional islands were then merged into this layer as requested. A shapefile was created from this final raster file to provide a clean version of this boundary file in vector format. This vector file is more efficient for drawing in the ArcGIS viewer as it is reduced in file size, opposed to the large original line work file. The final layers are projected in Geographic decimal degrees, WGS84 datum.
A line work quality check was completed using visible Modis imagery overlaying with our newly created Landsat dataset. It was determined that we could pick up any major quality errors with this smaller scale imagery with minimal image processing.
